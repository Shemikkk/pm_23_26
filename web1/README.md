# web1 — Gulp automation demo (pm_23_26)

This folder (`web1`) contains a small scaffold to demonstrate Gulp-based automation for frontend tasks: compiling SCSS, minifying CSS/JS, optimizing images, and live-reload with BrowserSync.

What was added
- `package.json` — dev dependencies and npm scripts
- `gulpfile.js` — tasks for HTML (file-include), SCSS, JS, images, and BrowserSync
- `app/` — source files
  - `index.html` and partial in `app/html/partials/head.html`
  - `scss/style.scss`
  - `js/main.js`
  - `img/readme.txt` (placeholder)
- `dist/` will be generated by Gulp after the first build

How to use
1. Open a terminal in the `web1` folder.

PowerShell example:

```powershell
cd c:\Users\Maks\Desktop\pm_23_26\web1
npm install
npm run dev
```

- `npm install` installs dev dependencies listed in `package.json`.
- `npm run dev` runs the default Gulp task: builds files to `dist/` and starts BrowserSync with watch.

Build only
```powershell
npm run build
```

Control questions (short answers)
1) Що таке Gulp і для чого він використовується?
   - Gulp — task runner для автоматизації рутинних задач фронтенд-розробки (компіляція, мінімізація, оптимізація, live-reload).
2) Які основні задачі автоматизації можна виконати за допомогою Gulp?
   - Компіляція препроцесорів (SCSS), об'єднання і мінімізація JS/CSS, оптимізація зображень, копіювання файлів, live-reload.
3) Чим відрізняються препроцесори CSS від постпроцесорів?
   - Препроцесори (SCSS, Less) додають синтаксичні розширення і компілюються у CSS; постпроцесори (Autoprefixer, PostCSS) обробляють готовий CSS (додають префікси, оптимізують).
4) Як працює BrowserSync і для чого його використовують?
   - BrowserSync запускає локальний сервер і синхронізує сторінки/перезавантаження між браузерами при зміні файлів.
5) Чи вимагається наявність на комп’ютері Node.js?
   - Так — Node.js і npm потрібні для встановлення Gulp та плагінів.
6) Яка структура папок у web-проекті є заданою за правилами хорошого тону web-розробки?
   - Наприклад: `app/` для джерел, `dist/` для збірки; підпапки: `scss`, `js`, `img`, `html`.
7) Функціональне призначення `package.json`?
   - Зберігає метадані про проєкт, список залежностей і скриптів для npm.
8) Функціональне призначення `gulpfile.js`?
   - Містить визначення тасків Gulp (що і як обробляти) і їх експорти.
9) Опис поняття "gulp task".
   - Окрема функція/завдання у `gulpfile.js`, яка визначає набір операцій над файлами (src -> pipe plugins -> dest).
10) Складові частини gulp таска.
   - Вхідні файли (`src()`), ланцюг обробки (`pipe()` з плагінами) і вихід (`dest()`); додатково — трансформації, помилки, стріми.
